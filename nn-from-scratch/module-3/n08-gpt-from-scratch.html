GPT From Scratch

Assembling self-attention, feed-forward layers, and positional encoding into the full transformer block

~15 min
Anchor Analogy
Think of assembling a car. Attention heads are the steering mechanism — they dynamically route information to where it is needed. Feed-forward layers are the engine — they do the heavy processing. Positional encoding is the GPS — without it, the car has no sense of where it is on the road. Each component is necessary. Remove any one and the vehicle does not work.
Why This Matters

Building a working GPT from scratch is the clearest possible demonstration that language models are not magic — they are specific engineering choices stacked together. When you build the architecture yourself, every design decision becomes visible. You see exactly why causal masking is necessary, why positional encoding cannot be skipped, and what role the feed-forward layer plays that attention alone cannot fill. This understanding transfers directly to reading research papers, debugging model behavior, and making architectural decisions in practice.

The Problem Attention Alone Cannot Solve

Attention is powerful, but it has a narrow job. It routes information — it decides which tokens should influence each other and how strongly. What it does not do is process that information once it has been routed.

Think of a management consulting team. Attention is the partner who decides which analysts need to talk to which client stakeholders. The right conversations are connected. But conversations alone do not produce a deliverable. Someone still needs to sit down and do the analysis, write the slides, draw the conclusions. That work happens after the routing.

In a GPT block, the feed-forward layer is the analysis work. After attention routes information from relevant tokens into each position, the feed-forward layer processes it — applying a learned nonlinear transformation that extracts patterns, compresses signals, and prepares the representation for the next layer.

This division of labor is deliberate. Attention operates across positions: it mixes information from the whole sequence. The feed-forward layer operates independently at each position: it takes what attention assembled and transforms it without looking at other positions. Together they form a complete processing unit.

The Full GPT Block: Four Operations in Order

A single GPT block contains exactly four operations. They stack in a fixed order, and each subsequent block repeats the same four operations. GPT-2 uses 12 of these blocks. GPT-3 uses 96. The stack is always the same pattern — only the depth and the width change (Brown et al., 2020).

The four operations, in order:

  1. Layer normalization before attention
  2. Causal multi-head attention
  3. Layer normalization before the feed-forward layer
  4. Feed-forward network with a nonlinear activation

Both attention and the feed-forward sublayer sit inside a residual connection — the block adds its output to its input rather than replacing it. The input flows through the block, accumulates modifications, and continues to the next block carrying both the original signal and the newly computed additions.

Why Residual Connections?

Residual connections solve a training problem that plagued deep networks before they were invented (He et al., 2016). When you stack many layers, gradients must flow backward through all of them. In a deep network without residual connections, the gradient signal degrades as it flows back — multiplied by many small numbers at each layer, it can vanish to near zero before reaching the early layers. Those early layers stop learning.

A residual connection provides a direct highway for the gradient. The shortcut path from the output of the block back to its input means the gradient can bypass all the internal computations and flow back cleanly. The block still learns its contribution, but the gradient for earlier layers no longer depends on the block cooperating.

In PyTorch:

# Residual connection pattern
x = x + self.attention(self.norm1(x))   # attention sublayer
x = x + self.feed_forward(self.norm2(x))  # feed-forward sublayer

Notice that normalization happens before each sublayer (pre-norm). This is the convention GPT-2 and later models use, and it produces more stable training than normalizing after the sublayer (Radford et al., 2019).

Causal Masking: The Key to Language Generation

Attention, as you built it in N07, lets every token see every other token. That works for tasks like translation, where you see the full source sentence before generating any output. It does not work for language generation, where you must predict the next token using only the tokens that came before.

If you are generating the fifth word of a sentence and the attention mechanism lets that position look ahead at the sixth, seventh, and eighth words, the model has access to the answer before it needs to produce it. During training it would learn to cheat. At inference time, those future tokens simply do not exist yet — so the cheat collapses and the model fails.

Causal masking prevents this. Before computing the softmax in attention, you apply a mask that forces every position to ignore positions that come after it in the sequence. The mask is a triangular matrix: position 0 sees only position 0, position 1 sees positions 0 and 1, position 2 sees positions 0, 1, and 2, and so on.

import torch

def causal_mask(seq_len):
    # Upper triangle (future positions) set to -infinity
    # Lower triangle (past positions) set to 0
    mask = torch.triu(torch.ones(seq_len, seq_len), diagonal=1)
    mask = mask.masked_fill(mask == 1, float('-inf'))
    return mask  # Shape: (seq_len, seq_len)

# In attention, add the mask to scores before softmax
scores = scores + causal_mask(seq_len)
attn_weights = torch.softmax(scores, dim=-1)

When softmax receives a very large negative value, it outputs a value extremely close to zero. So positions after the current one contribute nothing to the weighted sum — they are effectively invisible.

Think first: Why use −infinity in the mask rather than simply setting those attention weights to zero directly? +

Because the mask is applied to the raw attention scores, before the softmax. Softmax is a normalization operation — it converts a vector of scores into a probability distribution that sums to 1. If you set some weights to zero after softmax, you break the normalization: the remaining weights still sum to 1, but the computation that produced them included the now-zeroed positions. The model would have distributed attention incorrectly before you zeroed them.

Adding −infinity before the softmax means those positions receive near-zero attention after normalization, and the remaining positions receive correctly normalized weights — as if the masked positions never existed. The softmax remains mathematically sound.

Positional Encoding: Teaching the Model Where It Is

Here is a feature of the attention mechanism that is easy to overlook: it is permutation-invariant. If you shuffle the tokens in a sentence, the attention scores rearrange accordingly, but the model has no direct knowledge that the order changed. It sees the same set of tokens, related to each other in the same ways, just in a different arrangement.

This matters because language depends on order. "The dog bit the man" and "The man bit the dog" contain identical tokens. Their meaning is opposite. A model that is blind to position cannot distinguish them.

Positional encodings inject order information directly into the token representations. Before any attention is computed, each token's embedding is modified by adding a position-dependent signal. Now when the model processes "dog" in position 1 versus position 3, the representation it sees is different — position is encoded into the very numbers the model operates on.

Two Approaches: Learned vs. Fixed

GPT models use learned positional embeddings: a second lookup table, exactly like the token embedding table, but indexed by position rather than token ID. Position 0 has a learned vector. Position 1 has a different learned vector. These vectors are parameters trained alongside the rest of the model (Radford et al., 2018).

The original transformer paper used fixed positional encodings based on sine and cosine functions of varying frequencies (Vaswani et al., 2017). The principle is the same: add a position-dependent signal to each embedding. The difference is whether those signals are designed in advance or learned from data.

class GPTEmbedding(nn.Module):
    def __init__(self, vocab_size, context_len, d_model):
        super().__init__()
        self.token_emb = nn.Embedding(vocab_size, d_model)
        self.pos_emb = nn.Embedding(context_len, d_model)

    def forward(self, x):
        # x: (batch, seq_len) token indices
        B, T = x.shape
        positions = torch.arange(T, device=x.device)  # [0, 1, 2, ..., T-1]
        tok = self.token_emb(x)        # (B, T, d_model)
        pos = self.pos_emb(positions)  # (T, d_model)
        return tok + pos               # broadcasts across batch dimension

The addition is the key operation. Each token's semantic meaning (from the token embedding) and its location in the sequence (from the positional embedding) are combined into a single vector. The model then operates on these combined vectors, with both kinds of information encoded in the same representation.

The Feed-Forward Sublayer

After attention, each position holds a vector that blends information from relevant parts of the sequence. The feed-forward sublayer transforms this vector independently at each position. No cross-position communication happens here — each position is processed on its own.

The architecture is a standard two-layer MLP with a nonlinear activation between them. The hidden dimension is expanded significantly relative to the model dimension, then contracted back. GPT-2 expands by a factor of 4: a model with dimension 768 has a feed-forward hidden size of 3072 (Radford et al., 2019).

class FeedForward(nn.Module):
    def __init__(self, d_model, expansion=4):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(d_model, d_model * expansion),  # expand
            nn.GELU(),                                 # nonlinearity
            nn.Linear(d_model * expansion, d_model),  # contract
        )

    def forward(self, x):
        return self.net(x)  # applied independently to each position

Why expand and contract? The expansion creates a larger representational space where the model can separate and process features that are entangled in the original dimension. The nonlinear activation allows the model to learn discontinuous, non-proportional relationships. The contraction compresses the result back to the model's working dimension.

Research into what these layers do suggests they act as memory stores for factual associations (Geva et al., 2021). When the model recalls that "Paris is the capital of France," this association is more likely stored in the feed-forward weights than in the attention mechanism. Attention identifies that the question is about Paris; the feed-forward layer retrieves what it knows about Paris.

GELU vs. ReLU

GPT uses the GELU activation rather than the more common ReLU. ReLU simply sets negative values to zero: values either pass through or are eliminated. GELU smoothly attenuates small values rather than hard-cutting them at zero. In practice, GELU produces slightly better performance on language tasks, likely because the smooth boundary allows more nuanced gradient flow during training (Hendrycks & Gimpel, 2016).

Putting the Full Block Together

With all components defined, the GPT block is straightforward to assemble:

import torch
import torch.nn as nn

class GPTBlock(nn.Module):
    def __init__(self, d_model, n_heads, context_len):
        super().__init__()
        self.norm1 = nn.LayerNorm(d_model)
        self.attn = CausalMultiHeadAttention(d_model, n_heads, context_len)
        self.norm2 = nn.LayerNorm(d_model)
        self.ff = FeedForward(d_model)

    def forward(self, x):
        # Attention sublayer with residual
        x = x + self.attn(self.norm1(x))
        # Feed-forward sublayer with residual
        x = x + self.ff(self.norm2(x))
        return x


class GPT(nn.Module):
    def __init__(self, vocab_size, context_len, d_model, n_heads, n_layers):
        super().__init__()
        self.embedding = GPTEmbedding(vocab_size, context_len, d_model)
        self.blocks = nn.Sequential(
            *[GPTBlock(d_model, n_heads, context_len) for _ in range(n_layers)]
        )
        self.norm = nn.LayerNorm(d_model)
        self.head = nn.Linear(d_model, vocab_size, bias=False)

    def forward(self, x):
        x = self.embedding(x)       # token + position embeddings
        x = self.blocks(x)          # n_layers GPT blocks
        x = self.norm(x)            # final normalization
        logits = self.head(x)       # project to vocabulary
        return logits               # (batch, seq_len, vocab_size)

This is a complete GPT. You can train it with a cross-entropy loss on next-token prediction and it will work — not as powerfully as GPT-3, because it has far fewer parameters and far less data, but architecturally identical.

The final nn.Linear layer (called the language model head) projects from the model's internal dimension to a score for every token in the vocabulary. The token with the highest score is the model's prediction for the next token. During training, the correct next token provides the supervision signal. During generation, you sample from these scores (or take the highest) and append the result to the sequence.

The Training Loop

Training a language model is straightforward once the architecture is in place. You feed in a sequence of tokens, shift the targets by one position, and train the model to predict each next token:

def train_step(model, optimizer, x, criterion):
    # x: (batch, seq_len) of token indices
    inputs = x[:, :-1]    # all tokens except the last
    targets = x[:, 1:]    # all tokens except the first (shifted by 1)

    optimizer.zero_grad()
    logits = model(inputs)           # (batch, seq_len-1, vocab_size)
    loss = criterion(
        logits.reshape(-1, logits.size(-1)),  # flatten: (batch * seq_len-1, vocab_size)
        targets.reshape(-1)                   # flatten: (batch * seq_len-1,)
    )
    loss.backward()
    optimizer.step()
    return loss.item()

Every position in the sequence simultaneously contributes a training signal. A 512-token sequence provides 511 supervised examples per forward pass — one per next-token prediction. This is why language modeling is such an efficient use of data: the supervision signal is dense and free.

Key Insight

A GPT block is a two-phase operation: gather then process. Attention gathers relevant information from across the sequence into each position's representation. The feed-forward layer processes that gathered information at each position independently. These two phases are not interchangeable and cannot replace each other. Stacking them in alternation — block after block — creates a network that progressively builds richer, more context-aware representations from the raw token sequence.
