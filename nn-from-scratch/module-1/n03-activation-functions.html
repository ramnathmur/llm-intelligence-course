Activation Functions

The nonlinear gates that let networks learn curves not just lines.

~15 min
Anchor Analogy
A committee decision rule. Sigmoid says vote proportionally to conviction. ReLU says vote only if conviction exceeds a threshold — otherwise stay silent. The threshold rule produces sparser, faster-training networks.
Why This Matters
Activation function choice shapes what a network can learn and how fast it trains. ReLU’s dominance over sigmoid and tanh is not arbitrary — it solved the vanishing gradient problem that stalled deep networks for years. Understanding this choice is understanding why modern deep learning became possible at all.

Prerequisites: N02 — MLP From Scratch

Imagine a consulting firm that runs all major decisions through a committee of specialists. Each specialist scores a proposal from −10 to +10. If you simply add up those scores, the final recommendation is a weighted sum — a completely linear process. No matter how many rounds of committee review you run, the overall result is still just a weighted average of the original inputs. Adding more rounds does not let the committee handle more complex problems. It just moves numbers around.

That is the fundamental problem with a neural network that uses no activation functions, or only linear ones. Stack as many layers as you like and you still get a linear function of the original inputs. The whole enterprise collapses into a single matrix multiplication. To make a network genuinely deeper — to let it learn hierarchical, curving, non-straight relationships between inputs and outputs — you need something that breaks the linearity. That something is the activation function.

An activation function sits at the output of every neuron. It takes the neuron’s raw score (the weighted sum of its inputs) and transforms it before passing the result to the next layer. The transformation is deliberately nonlinear: it introduces bends, kinks, and thresholds. Those bends are what allow deep networks to carve up complex decision boundaries, approximate curves, and ultimately learn to recognise handwriting, translate languages, and generate code.

Historically, three activation functions have dominated: sigmoid, tanh, and ReLU. They each represent a different theory of what that nonlinear gate should do. The history of which one won — and why — is also the history of how the field learned to train genuinely deep networks.

Sigmoid: the original probabilistic vote

Sigmoid was the default activation function for the first generation of neural networks. It has a clean probabilistic story: it squashes any real number into the interval (0, 1), which you can read as a probability. A neuron with sigmoid activation is asking: given this weighted evidence, how strongly do I vote “yes”?

The formula looks like this: σ(x) = 1 / (1 + e−x). For large positive inputs it approaches 1. For large negative inputs it approaches 0. At zero it returns exactly 0.5. The curve is S-shaped — gently rising in the middle, flattening out at the extremes.

The three major activation functions, compared side by side. Hover over each curve to see key properties. The x-axis is the neuron’s raw input; the y-axis is what gets passed forward.

The elegance of sigmoid has a hidden cost. Look at what happens at the extremes of the curve. When the input is a large positive number, the output is stuck very close to 1. When the input is a large negative number, the output is stuck very close to 0. In both cases the curve is nearly flat. A nearly flat curve has a nearly zero slope. And slope is exactly what backpropagation uses to push learning signals backwards through the network.

Why flat regions kill learning: the vanishing gradient

To understand why flatness is catastrophic, you need a quick picture of how learning signals travel through a neural network. During backpropagation — explored in detail in N02 — the network computes how much each weight contributed to the error. It does this by multiplying gradients backwards through each layer, from the output layer towards the input layer.

Here is the problem. Each time the signal passes through a layer with sigmoid activation, it gets multiplied by the slope of the sigmoid at that layer. When that slope is small — which it is whenever the neuron has saturated to a value near 0 or 1 — the signal shrinks. Pass through ten layers, each multiplying by a small number, and you are multiplying small numbers together ten times. The result is a gradient so tiny it is effectively zero.

The layers closest to the input receive an almost-zero learning signal. From those layers’ point of view, the network has stopped learning. Weights barely move. Training stalls. The features learned in early layers are essentially frozen and never improved. This is the vanishing gradient problem, and it is the reason that for most of the 1990s and 2000s, neural networks with more than a few layers simply refused to train (Hochreiter, 1991).

Think Further

If you multiply ten numbers each equal to 0.25 together, you get 0.2510 ≈ 0.000001. That is roughly a million times smaller than the original signal. Now consider that a practical deep network might have dozens of layers. The gradient does not just shrink — it vanishes to a value indistinguishable from floating-point noise.

Tanh: a partial fix with the same flaw

Tanh (hyperbolic tangent) was the next widely used activation function. It looks like a stretched sigmoid, but with output range (−1, +1) instead of (0, 1). The zero-centred output was a genuine improvement: because outputs average to zero rather than 0.5, the gradient updates in the next layer tend to be less biased and training tends to be faster in practice.

But tanh shares sigmoid’s fundamental structural problem. It also has flat regions at both extremes. Neurons that saturate to +1 or −1 produce nearly-zero gradients. The vanishing gradient problem persists. Tanh is a better sigmoid, not a different kind of solution.

ReLU: the kink that fixed deep learning

In 2010 and 2011, a series of papers demonstrated that a radically simpler function outperformed sigmoid and tanh for training deep networks. The Rectified Linear Unit — ReLU — is almost embarrassingly simple: output the input if it is positive, otherwise output zero (Nair & Hinton, 2010). In Python: max(0, x).

That is the entire function. No exponentials. No divisions. One comparison and one multiplication per neuron. Yet this function solved the vanishing gradient problem in a single stroke, and it is the reason you can train networks with hundreds of layers today.

Why ReLU does not vanish

For any input greater than zero, ReLU’s slope is exactly 1. Not small, not shrinking — exactly 1. When backpropagation passes through a ReLU neuron that is active (receiving a positive input), it multiplies the gradient by 1, which means the gradient flows through unchanged. Stack a hundred active ReLU layers and the gradient from the output reaches the first layer just as strong as it left. No multiplication chain of small numbers. No vanishing.

This is the “threshold vote” from the opening analogy made mathematical. A ReLU neuron either votes with full conviction (gradient = 1) or stays completely silent (gradient = 0). There is no “mostly convinced” state that attenuates the signal.

Think first: what does it mean for a ReLU neuron’s gradient to be exactly 1 when active?+

It means the neuron is a perfect gradient highway. Whatever learning signal arrives at its output gets passed backwards to its inputs without any scaling. The neuron contributes to learning exactly as much as the downstream error dictates — no more, no less. Compare this to sigmoid, where even a lightly-saturated neuron might pass only 10% of the signal through, compounding across every layer until nothing reaches the early weights.

A second benefit is computational. Training a large network requires evaluating millions of activations per second. Sigmoid requires computing an exponential function for every neuron on every forward and backward pass. ReLU requires only a comparison: is this number positive? On modern hardware, the difference in speed is substantial. ReLU enabled networks to grow larger and train faster simultaneously, which is why the deep learning renaissance that began with AlexNet in 2012 — which used ReLU throughout — would not have been practical with sigmoid (Krizhevsky et al., 2012).

The dead neuron problem: ReLU’s cost

ReLU is not a free lunch. Look at the left half of its curve in the diagram above. For any input less than or equal to zero, ReLU outputs exactly zero. The gradient through that region is also exactly zero. A neuron that consistently receives negative inputs will output zero on every forward pass and receive zero gradient on every backward pass. Its weights never update. The neuron is effectively switched off for the rest of training. It has become a dead neuron.

How does a neuron get stuck like this? Two common routes. First, the initial random weights might be configured such that the neuron’s input is always negative from the first batch forward. Second, and more insidiously, a very large gradient update can push the neuron’s weights so negative that no subsequent training example can revive it. This is called the “dying ReLU” problem, and in poorly-tuned networks a significant fraction of neurons can die and stay dead, wasting network capacity.

A neuron in your network has been receiving negative weighted sums for the last 10,000 training steps and its weights have not changed in that time. What is most likely happening?

Leaky ReLU: patching the dead zone

Leaky ReLU was proposed as a simple patch for the dead neuron problem (Maas et al., 2013). Instead of outputting exactly zero for negative inputs, it outputs a small positive fraction α of the input (typically 0.01). The formula is: max(αx, x).

This tiny leak keeps the gradient flowing. For negative inputs, the gradient is no longer zero — it is α, a small but non-zero number. A neuron that would have died with standard ReLU can now still receive a tiny learning signal and eventually recover if conditions change.

What does Leaky ReLU trade away? Two things. First, the clean “sparse activation” property of ReLU disappears. With ReLU, neurons are either fully on or fully off, which means most neurons are silent for any given input — a useful inductive bias for many problems. With Leaky ReLU, all neurons always have at least a small output, increasing the density of activations throughout the network.

Second, α is a hyperparameter. Set it too large and you lose the benefit of having a threshold at all; the function approaches a linear mapping. Set it too small and you do not fix the dead neuron problem meaningfully. Practitioners must tune this, and different datasets favour different values. Parametric ReLU (PReLU) takes this further by making α a learnable weight rather than a fixed constant (He et al., 2015), adding flexibility at the cost of more parameters.

Standard ReLU

Gradient is exactly 1 for active neurons. Dead neurons receive zero gradient and can never recover. Activations are sparse. No hyperparameter to tune. Fast to compute.

Leaky ReLU

Gradient is α for negative inputs — small but non-zero. Dead neurons can recover over training. Activations are less sparse. Requires tuning α. Marginally slower.

The modern landscape: ELU, GELU, Swish

The story did not stop with Leaky ReLU. As researchers built ever-deeper networks for increasingly diverse tasks, new activation functions emerged that offered different trade-offs.

ELU (Exponential Linear Unit) uses an exponential curve in the negative region rather than a flat line or a leak (Clevert et al., 2016). This makes the average output close to zero (like tanh) while preserving non-zero gradients for negative inputs (like Leaky ReLU). The result tends to be faster convergence on some architectures, at the cost of the exponential computation being slower than a simple comparison.

GELU (Gaussian Error Linear Unit) is the activation function used in BERT, GPT, and most modern transformer architectures (Hendrycks & Gimpel, 2016). It is a smooth approximation to ReLU that gates the input by a probability derived from its position in a Gaussian distribution. Intuitively, GELU is asking: “what is the probability that a random noise perturbation would not zero this input out?” Inputs with high probability of being meaningful get passed through at full strength. Borderline inputs get attenuated. GELU avoids the hard kink that ReLU has at zero, which helps training on some tasks, particularly in language models where that kink can cause instability.

Swish (also called SiLU) is defined as x · σ(x) — the input times its own sigmoid (Ramachandran et al., 2017). It was found by automated neural architecture search, not by human intuition. Like GELU, it is smooth and non-monotonic — it dips slightly below zero for small negative inputs before flattening. Swish consistently outperforms ReLU on deep residual networks and is the default in EfficientNet and many production vision models.

Key Insight

Every activation function is a different answer to the same question: how should a neuron gate its output? Sigmoid gates proportionally (everything gets through, scaled by conviction). ReLU gates by threshold (all-or-nothing, silence below zero). Leaky ReLU allows a whisper below the threshold. GELU and Swish gate by a smooth probability. The gradient properties that flow from each answer determine whether deep networks train at all — and ReLU won not because it is the most elegant, but because it is the one whose gradients do not shrink.

How to choose in practice

Given the range of options, practitioners follow a simple decision ladder.

Start with ReLU. It is fast, well-understood, and works on the majority of problems including image classification, regression, and basic sequence tasks. If you are building a network from scratch and you do not have a reason to choose otherwise, ReLU is the correct default.

Switch to Leaky ReLU or ELU if you observe dead neurons. You can detect dead neurons by monitoring the fraction of neurons outputting zero on a validation batch. If a significant portion (say, more than 10%) is consistently zero, the dying ReLU problem is active and Leaky ReLU is a straightforward fix.

Use GELU for transformer-based architectures. Language models and vision transformers almost universally use GELU because it matches the smooth, probabilistic nature of attention-weighted computation better than the hard threshold of ReLU. If you are building on top of a pre-trained transformer backbone, match its activation function to avoid mismatch.

Consider Swish for deep convolutional networks. On very deep residual networks, Swish and its close cousin Mish often converge faster and to better final accuracy than ReLU. The cost is a more expensive forward pass, which matters if you are training on a tight compute budget.

Sigmoid and tanh have their place at outputs, not in hidden layers. For binary classification, a single sigmoid output neuron is still standard: it produces a probability between 0 and 1. For recurrent networks (LSTMs and GRUs), tanh is used within the gating mechanisms for specific mathematical reasons. In hidden layers, avoid both for any network deeper than two or three layers.

Connection Forward

The activation function choice you make in a feedforward network carries directly into every advanced architecture. When you build convolutional networks, the same ReLU layers appear after each filter. When you read about transformer attention in the LLM Intelligence course, the feed-forward sublayer inside each transformer block is two linear layers with GELU between them — exactly the pattern from this page, just repeated 96 times in a large model. The principle scales with architecture but does not change.

The sparsity bonus: why dead neurons are not purely bad

There is one more angle worth holding. The fact that ReLU silences neurons for negative inputs is not only a risk (dead neurons). It is also a feature. When a network uses ReLU, typically only a fraction of neurons are active on any given input. The representation of a specific image, or a specific sentence, is encoded in a sparse subset of all the neurons in the network.

Sparse representations have several advantages. They are easier to separate: two different inputs that activate different sparse subsets are automatically distinct, even if the overall network is large. They tend to be more robust: corrupting a few neurons in a sparse code degrades the representation less than corrupting the same proportion of a dense code. And there is a biological parallel: cortical neurons in the mammalian brain are thought to operate in a sparse regime, with roughly 5–10% active at any given time. Whether that parallel is meaningful or coincidental is debated, but the engineering advantages of sparsity are well-established in the deep learning literature (Glorot et al., 2011).

The dead neuron problem is a bug. The sparse activation pattern is a feature. They arise from the same cause — ReLU zeroing negative inputs — and good network design tries to capture the benefit while avoiding the failure mode. Learning rate scheduling, careful weight initialisation (especially He initialisation, designed specifically for ReLU networks), and batch normalisation all help keep most neurons alive while still enjoying the sparsity benefit.
