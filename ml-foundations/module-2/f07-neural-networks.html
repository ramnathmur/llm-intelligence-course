Neural Networks as Stacked Functions

A neural network is a chain of linear transformations with nonlinear gates between them.

~15 min
Anchor Analogy
A series of consultants each transforming a client brief — each one takes the previous consultant's output, applies their specialist lens, and passes a transformed document forward. The final consultant hands you a recommendation, not a summary of the original brief.
Why This Matters
The neural framing is a historical accident. Thinking of a neural network as stacked matrix multiplications with activation functions is both more accurate and more useful — it makes the architecture legible. Once you see a neural network as a composed function rather than a brain metaphor, you can reason about what each layer does, why depth matters, and what goes wrong when a network fails to learn.
After This Page, You Can Argue
What a neural network is mechanically, why nonlinearity is essential, and what depth adds that width alone cannot provide. You will also be able to explain why stacking linear layers without activation functions is equivalent to a single linear layer, and why that makes the "neural" name something of a misnomer.
Prerequisites: F03 — Gradient Descent

The name "neural network" conjures images of a simulated brain: neurons firing, synaptic connections, biological inspiration. That framing is more hindrance than help. It suggests something organic, mysterious, and hard to reason about.

Strip the metaphor away and what you have is a function. A composed function, built by chaining simpler functions together. Each stage in the chain takes a set of numbers, applies a learnable transformation, and passes the result to the next stage. The final stage produces the prediction. Everything else is detail.

This page builds the mechanical picture from scratch. By the end, "neural network" will mean something precise to you — not an analogy to biology, but a specific class of mathematical object with known properties and known limitations.

Start with a single layer: the linear transformation

The fundamental operation in a neural network is a linear transformation. You give it a list of input numbers. It multiplies each input by a learned weight, sums everything together, and adds a learned offset called a bias. The output is a single number — or, if the layer has multiple outputs, a list of numbers produced by running this operation once for each output.

The consulting analogy makes this concrete. A junior analyst receives a client brief that contains fifty data points — revenue, headcount, market share, churn rate, and so on. Their job is to synthesise those fifty numbers into five strategic signals: growth momentum, cost efficiency, competitive position, customer health, and execution risk. They apply a learned weighting: revenue matters more for growth momentum; churn rate matters more for customer health. The result is five synthesised scores from fifty inputs.

That is a linear layer. Fifty inputs, five outputs, learned weights, one bias per output. In code it is often written as a matrix multiplication: the weight matrix transforms the input vector into the output vector (Goodfellow et al., 2016).

A linear layer is learnable — training adjusts the weights. But on its own it has a severe limitation: it can only represent linear relationships. If you double the inputs, the outputs double. If you add two inputs together before passing them in, the output changes additively. The whole world of curves, thresholds, interactions, and non-proportional relationships is invisible to a single linear layer.

The nonlinearity problem: why linear alone is not enough

Here is the critical mathematical fact: if you stack multiple linear transformations on top of each other without putting anything between them, the composition is still a linear transformation. It does not matter how many layers you add. The result is always equivalent to a single matrix multiplication with different numbers.

Think first: why does stacking two linear layers collapse to one?+

A linear transformation is defined by a matrix. Composing two linear transformations means multiplying their matrices together. Matrix multiplication produces another matrix — and that resulting matrix is a linear transformation in its own right. No matter how many matrices you multiply, the product is always a matrix. So ten stacked linear layers with no activation functions in between are mathematically identical to a single linear layer with a different set of weights. You have added parameters but not added any representational power. This is why depth without nonlinearity is a fiction: you are not building a deeper model, you are just parameterising a single linear map in a roundabout way.

This collapse means a network of any depth with only linear operations can only draw straight-line (or flat-plane) decision boundaries. It can learn that sales correlates linearly with marketing spend. It cannot learn that the relationship is strong at low spend levels, saturates at medium spend, and reverses at very high spend (ad fatigue). Curves, kinks, and interactions are permanently out of reach.

Most interesting problems in the world involve exactly those non-linear patterns.

The fix: activation functions as nonlinear gates

The solution is simple and elegant: after each linear layer, apply a nonlinear function to the outputs before passing them to the next layer. This function is called an activation function.

The activation function takes each output number from the linear layer and applies a fixed mathematical operation to it, independently, without involving the other outputs. It introduces a bend into the otherwise straight-line path from input to output. And that bend is what lets the network learn curves.

The most popular modern activation function is the ReLU (Rectified Linear Unit) (Nair & Hinton, 2010). Its rule is almost embarrassingly simple: if the input is negative, output zero; if the input is positive, output the input unchanged.

In the consulting analogy: after the junior analyst produces their five strategic scores, a senior reviewer applies a threshold. Any signal that comes in below a certain level is treated as "not material" and zeroed out. Only signals above the threshold are passed forward with their full magnitude. The threshold is fixed; the weighting happens before and after it. But that threshold — that gate — is what makes the whole chain non-linear.

ReLU is a gate with a kink at zero. Negative inputs are zeroed out; positive inputs pass through unchanged. That single kink is enough to break the linear collapse — compose enough layers with ReLU gates and the network can approximate arbitrarily complex functions.

Other activation functions exist. The sigmoid function squashes any input into the range (0, 1), which made it popular in the early days for producing probability-like outputs. The tanh function squashes into (−1, 1). The GELU function, used in modern language models, is a smooth approximation to ReLU with slightly better empirical properties (Hendrycks & Gimpel, 2016). But the logic is always the same: introduce a bend so the linear collapse cannot happen.

Stacking layers: what depth actually does

Now you have the two ingredients: a linear transformation that mixes inputs into outputs via learned weights, and a nonlinear activation that introduces a bend. Stack these alternating layers and you have a deep neural network.

Each pair of (linear layer + activation) is called a hidden layer. "Hidden" just means it is between the input and the output — not directly observed by the user, but doing real work internally.

Why stack multiple layers rather than just one? The key insight is that each layer builds on what the previous layer learned.

Return to the consulting chain. The junior analyst synthesises fifty raw data points into five strategic signals. A mid-level consultant takes those five signals and synthesises them into two higher-order judgements: "structural health" and "strategic momentum." A senior partner takes those two and produces a single recommendation: "acquire" or "restructure." At each level, the human is working with abstractions built by the level below. The raw data never reaches the senior partner; they work with processed signals that carry more meaning than any individual number.

Neural network layers work the same way. Early layers learn to detect simple patterns in the input: in image recognition, they detect edges and colour gradients. Middle layers combine those simple patterns into more complex ones: textures, object parts. Later layers combine those intermediate representations into high-level concepts: "this is a dog," "this is a face." The hierarchy is not specified by the engineer — it emerges from training, because hierarchical representations happen to be what minimises the loss (LeCun et al., 2015).

Without Depth (Single Layer)

One linear layer plus activation. The network can learn which input features matter, but it can only combine them in a single step. Complex interactions — where the importance of feature A depends on the value of feature B — are hard to represent. The decision boundary is simple.

With Depth (Multiple Layers)

Each layer reprocesses the outputs of the previous layer. Interactions among features are captured in intermediate representations that no individual input carries. The network can represent exponentially more complex functions with the same total number of parameters. The decision boundary can be arbitrarily complex.

What depth buys that width cannot

You might ask: instead of adding more layers, why not just make each layer wider — more outputs per layer, more parameters? This is the depth-versus-width question, and the answer is non-obvious but important.

Width helps a network approximate complex functions by sheer brute force. Given enough neurons in a single hidden layer, a network can theoretically approximate any continuous function to arbitrary precision — this is the universal approximation theorem (Hornik, 1991). But "theoretically" and "practically" are different things.

In practice, depth is far more parameter-efficient for the kinds of structured patterns that appear in natural data. Here is why: depth allows reuse of computations. A pattern recognised in layer two can be used by both layer three and layer four simultaneously. A wide single layer must re-learn every combination of features from scratch, with no sharing.

Think of a consulting firm's knowledge management system. A shallow but very wide system would require every consultant to independently re-derive every insight from raw data. A deep system (a hierarchy of analysts, consultants, and partners) lets higher levels reuse and build on the work of lower levels. The deep hierarchy achieves the same outcome with far fewer total work-hours because knowledge compounds up the chain rather than being rebuilt at every step.

Depth also makes it easier for gradient descent to find good solutions. Hierarchical structure provides a natural scaffold: each layer has a more tractable sub-problem than the overall function, and solving each sub-problem contributes to solving the whole. Very wide shallow networks have flat, less structured loss landscapes that are harder to optimise.

Key Insight

Width lets a network represent complex patterns by having enough neurons. Depth lets a network build complex patterns by composing simpler ones. The same representational power achieved with depth requires exponentially more neurons with width. This is the practical reason deep networks outperform wide shallow ones on structured problems in the real world.

The full picture: a network is a composed function

Let us make the mechanical picture explicit. A three-layer neural network applies the following chain of operations to its inputs:

Layer 1: multiply inputs by weight matrix W1, add bias b1, apply activation → hidden representation h1

Layer 2: multiply h1 by W2, add bias b2, apply activation → hidden representation h2

Layer 3 (output): multiply h2 by W3, add bias b3 → prediction

The output layer often omits the activation (for regression tasks where you want the raw number) or applies a specific output activation such as softmax (which converts raw scores into a probability distribution over classes).

All the W and b values are parameters. Training adjusts them to minimise prediction error on the training data, via gradient descent (see F03 — Gradient Descent). The architecture — how many layers, how wide each layer, which activation functions — is a design choice the engineer makes before training begins.

That is it. No biological neurons. No firing thresholds. No synaptic plasticity. Just a composed function with learnable parameters and nonlinear gates between layers.

Why the "neural" name persists despite being misleading

The biological framing came from the field's origin in the 1940s, when McCulloch and Pitts proposed a mathematical model of a neuron (McCulloch & Pitts, 1943). The term "neural" stuck through decades of subsequent work. By the time practitioners recognised that the biological analogy was doing more harm than good, the vocabulary was too entrenched to change.

The harm is real: when people think "simulated brain," they attribute properties to neural networks that they do not have (self-awareness, understanding, genuine intelligence) and miss properties they do have (systematic approximation of functions, compositional representations, sensitivity to training distribution).

When a practitioner says "neural network," hear "composed function with learnable linear layers and nonlinear activations." The architecture diagrams with circles and arrows are just visual shorthand for matrix multiplications. The weights are just numbers. The training is just gradient descent minimising a loss.

Reading an architecture diagram

Architecture diagrams draw each layer as a column of circles (nodes) and each weight as an arrow between circles. When you see this, the translation is:

Each circle is one output value from a layer. The arrows from all circles in the previous layer to one circle in the current layer represent one row of the weight matrix — the weights that combine all previous outputs to produce one current output. The activation function is applied after all the multiplication-and-summing happens, before the circle's value is passed to the next layer.

A "fully connected" or "dense" layer means every circle in layer N connects to every circle in layer N+1. The weight matrix has one parameter for every such connection. If layer N has 512 nodes and layer N+1 has 256 nodes, the weight matrix has 512 × 256 = 131,072 parameters for that one layer alone.

Modern networks have billions of parameters because they are deep (many layers) and wide (many nodes per layer). A large language model might have hundreds of layers, each processing thousands of values simultaneously. The core operation at each layer is always the same: a linear transformation, then a nonlinearity. Scale that operation up, repeat it many times, and you have a frontier model.

Think Further

Not all layers in modern networks are fully connected. Convolutional layers apply the same small weight matrix to every patch of an image, dramatically reducing the number of parameters while exploiting the spatial structure of images. Attention layers, used in transformers, compute a different kind of mixing where the weight matrix itself depends on the input. You will encounter both when you study transformer architecture. The core principle — linear transformation plus nonlinearity — remains, but the structure of the weight matrix changes to match the structure of the data.

Practical implications for practitioners

Understanding neural networks as stacked functions gives you specific diagnostic tools when something goes wrong.

If your network trains but never improves, and you suspect the activation function is the issue, you may be encountering the vanishing gradient problem: in deep networks with certain activations (particularly sigmoid and tanh), the gradient signal that propagates back through many layers can shrink to near-zero, making the early layers unable to learn. ReLU was widely adopted partly because it does not have this property for positive inputs — the gradient through a ReLU for positive inputs is simply 1, unchanged by the layer's presence.

If your network learns training data perfectly but fails to generalise, depth is probably not the issue — overfitting is (see F06 — Overfitting and Regularisation). Adding more layers typically makes overfitting worse, not better, by increasing model capacity.

If you need to choose between a wider or deeper network for a task, prefer depth when the data has hierarchical structure (images, text, audio, time series). Prefer width when the task requires combining many independent signals at a single level of abstraction (tabular data with many features that matter roughly equally).

If you are told that a model uses a "two-layer network" or a "five-layer network," check the convention being used. Some practitioners count only the hidden layers; others count input and output layers too. A "two-layer network" in one paper might mean one hidden layer; in another it might mean two hidden layers plus an output layer. Always check the architecture diagram rather than relying on the count.

Key Insight

A neural network is a function composed of alternating linear layers and nonlinear activations. The linear layers learn to mix and weight inputs. The activations introduce the bends that allow the composed function to represent curves, thresholds, and interactions. Remove the activations and the whole stack collapses to a single linear transformation. The architecture is legible once you see it this way: adding a layer adds one more level of abstraction, and each activation keeps those abstractions nonlinearly expressive.
